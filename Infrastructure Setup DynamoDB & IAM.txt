Step-by-Step Implementation

Step 1: Create DynamoDB Table
Navigate to AWS Console → Services → DynamoDB
Initialize Table Creation
Click Tables → Create table
Configure Table Settings
Table name: CoffeeShop
Partition key: coffeeId (String)
Sort key: Not required (keeps the design simple)
Create the Table
Leave other settings as default (AWS handles provisioning automatically)
Click Create table




Step 2: Add Test Data
Why Add Test Data? Creating a sample coffee item serves multiple purposes:

Validation: Confirms table structure is correct
Testing: Provides data for API testing
Development: Enables frontend development without empty states
Access Table Items
Click on CoffeeShop table name
Navigate to Explore items
Create Sample Coffee Item
Click Create item

Add the following attributes:

{
  "coffeeId": "c123",
  "name": "new cold coffee",
  "price": 456,
  "available": true
}


Save the Item
Click Create item
Note: This test item will be returned when we make API calls, allowing us to validate the complete data flow from DynamoDB → Lambda → API Gateway → Frontend.
